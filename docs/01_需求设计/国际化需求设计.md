# MemeHunter 页面国际化需求设计

> 为 MemeHunter 前端页面增加多语言支持，提升全球用户体验

---

## 一、问题背景

当前 MemeHunter 前端所有界面文本均为硬编码的英文字符串，例如：

- 首页: "CONNECT WALLET"、"Catch Memes, Win Airdrops on Monad"
- 游戏页: "Hunts"、"Catch"、"Earned"、"Exit" 等
- 组件: "Leaderboard"、"History"、"CAUGHT"、"ESCAPED" 等
- 弹窗: "Session Key Active"、"Authorize Session Key"

这种硬编码方式存在以下问题：
1. **无法支持多语言**：非英语用户体验较差
2. **文本分散**：修改文案需遍历多个文件
3. **不利于扩展**：未来添加新语言成本高

---

## 二、需求目标

### 2.1 功能需求

| 需求项 | 描述 | 优先级 |
|--------|------|--------|
| 多语言支持 | 支持英文 (en) 和中文 (zh) 两种语言 | P0 |
| 语言切换 | 用户可手动切换界面语言 | P0 |
| 语言持久化 | 用户选择的语言保存到 localStorage | P1 |
| 自动检测 | 首次访问根据浏览器语言自动选择 | P2 |

### 2.2 涉及页面

| 页面/组件 | 文件路径 | 主要文本内容 |
|-----------|----------|--------------|
| 首页 | `pages/Home.tsx` | 标题、按钮、状态提示 |
| 游戏页 | `pages/Game.tsx` | 统计信息、结果提示 |
| 控制栏 | `components/game/ControlBar.tsx` | 网尺寸名称、费用 |
| 排行榜 | `components/game/Leaderboard.tsx` | 标题、空状态提示 |
| 历史记录 | `components/game/HuntHistoryPanel.tsx` | 标题、状态文本 |
| 玩家栏 | `components/game/PlayerBar.tsx` | 等待提示、在线人数 |
| 授权弹窗 | `components/wallet/SessionKeyModal.tsx` | 说明文本、按钮 |

### 2.3 用户场景

**场景 1：首次访问**
- 用户打开网站
- 系统检测浏览器语言偏好
- 自动显示对应语言界面

**场景 2：手动切换语言**
- 用户点击语言切换按钮（位于页面右上角）
- 界面即时切换为目标语言
- 选择保存到 localStorage

**场景 3：再次访问**
- 用户再次打开网站
- 系统读取 localStorage 中的语言设置
- 显示上次选择的语言

## 七、讨论记录

### 讨论主题：i18n 技术选型

**产品经理**：需求明确，两种语言支持，用户可手动切换。

**架构师**：建议使用 `i18next` + `react-i18next`，这是 React 生态最成熟的国际化方案，文档丰富，社区活跃。

**测试工程师**：需要覆盖语言切换后所有页面文本正确显示，以及 localStorage 持久化。

**最终决策**：
- 使用 `react-i18next` 作为国际化方案
- 翻译文件使用 JSON 格式
- 语言切换组件放在首页和游戏页的顶部导航栏
